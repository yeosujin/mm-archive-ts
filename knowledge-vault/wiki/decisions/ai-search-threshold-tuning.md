# AI 검색 임계값을 절대 하한 + 상대 컷으로 자른다

> [!note] 사후 복원된 ADR
> 코드와 인라인 주석(`src/pages/Search.tsx:187-190`)에서 역추적했다. 값의 근거는 주석에 남아 있으나, 후보값을 어디까지 시도했는지는 기록이 없다.

## 맥락

벡터 검색은 **항상 무언가를 돌려준다.** 아무 관계 없는 질의에도 상위 N개가 나오고, 그대로 렌더하면 "AI 검색이 헛소리를 한다"로 읽힌다.

동시에 이 아카이브는 **한/영이 섞인 콘텐츠**를 다룬다. 교차언어 매칭은 같은 언어끼리보다 유사도가 구조적으로 낮게 나온다.

## 결정

두 값을 조합해 컷오프를 만든다.

```typescript
const aiCutoff = Math.max(AI_SIM_FLOOR /* 0.64 */, topSim - AI_SIM_MARGIN /* 0.05 */)
```

- `AI_SIM_FLOOR = 0.64` — 교차언어 매칭이 낮게 나오는 것을 감안해 **의도적으로 낮춘 값**이다
- `AI_SIM_MARGIN = 0.05` — 최고점 대비 상대 컷

## 기각한 대안

| 대안 | 기각 이유 |
|---|---|
| 절대 하한만 | 좋은 결과가 나온 질의에서도 애매한 꼬리가 함께 붙는다 |
| 상대 컷만 | 전부 무관한 질의에서 "최고점 근처"만 남아 그대로 노출된다 |
| 하한을 더 높게 (예: 0.75) | 한→영, 영→한 매칭이 통째로 사라진다 |
| 상위 N개 고정 | 관련 결과가 3개뿐일 때 무관한 7개가 따라온다 |

## 결과

- 무관한 질의에서 결과가 **0건**으로 나오는 경우가 생긴다. 이건 의도한 동작이고, 0건일 때의 재검색 유도 CTA로 보완한다
- 임베딩 모델을 바꾸면 유사도 분포가 달라져 **두 값을 다시 잡아야 한다**
- 값의 근거가 인라인 주석에만 있다. 주석을 지우면 근거가 사라진다
