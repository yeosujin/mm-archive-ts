# 설계: "그해 오늘" X 자동 게시 봇

- 작성일: 2026-07-15
- 상태: 설계 승인됨 (구현 계획 대기)

## 1. 목적

매일 한국시간 자정에, 오늘과 같은 월/일(MM-DD)의 과거 콘텐츠를 X(트위터)에
쓰레드로 자동 게시한다. mmemory 아카이브의 "그해 오늘" 콘텐츠를 팬들에게 매일 노출하는 게 목표.

부수 목표(개인): Node.js + 외부 API 연동(X v1.1/v2) + 크론 자동화 + Supabase/R2 연동 경험.

## 2. 스코프

게시 대상은 **올릴 파일이 실제로 존재하는 콘텐츠**만이다. 파일 실체는 전부 Cloudflare R2에 있다.

| 종류 | 소스 필드 | 미디어 |
|------|-----------|--------|
| 모먼트 (Moment) | `tweet_url` (R2 영상) | 영상 |
| 사진 (Photo) | `image_url` (R2 이미지) | 이미지 |
| 포스트 (Post) | `media[]` (R2 이미지/영상) | 이미지+영상 혼재 |

**제외**: 영상 메뉴(Video, 대부분 YouTube/Weverse라 R2 파일 아님), 에피소드, 도서관.
포스트 중 `media[]`가 비어 있는 임베드 전용 항목은 스킵.

## 3. 콘텐츠 선정

- Supabase에서 `moments`, `posts`, `photos`를 조회한다.
- 기존 `src/lib/dailyPick.ts`의 `filterOnThisDay(items, todayString)` 로직을 재사용한다
  (같은 MM-DD + 과거 연도만, 오늘 연도 제외).
- `todayString`은 **한국시간(KST) 기준 오늘 날짜**(`YYYY-MM-DD`)로 계산한다.
- 트윗 본문에 **URL을 넣지 않는다** (종량제에서 URL 포함 게시는 $0.20, 미포함은 $0.015).

## 4. 텍스트 규칙

각 트윗 본문 포맷은 `YYMMDD 제목` (6자리 날짜 + 공백 + 제목).

| 종류 | 텍스트 | 가공 |
|------|--------|------|
| 사진 | `YYMMDD 제목` | 제목 끝 subfix(`-1`, `-2` 등) 제거 (`생일-1` → `생일`) |
| 모먼트 | `YYMMDD 플랫폼` | 상위 영상(`video_id`) 플랫폼명(**한글**: 유튜브/위버스/인스타그램/트위터). 플랫폼이 '기타'면 `platform_name` 표시. **상위 영상이 없으면 `YYMMDD`만** |
| 포스트 | `YYMMDD 제목` | 제목은 선택 → 없으면 날짜만. 제목 앞에 연도 prefix가 있으면 제거 (`2025 생일` → `생일`) |

- 모먼트 플랫폼 판별: `detectVideoPlatform(parentVideo.url)` → 한글 라벨 매핑(`platform.ts`의 `PLATFORM_KO`); `'other'`면 `parentVideo.platform_name`.
- **모먼트 날짜/선정 기준(홈 일치)**: 상위 영상이 있으면 **그 영상의 `date`** 로 "그해 오늘" 판정 및 `YYMMDD` 라벨링(모먼트 자체 날짜가 달라도 상위 영상 날짜로 포함). 독립 모먼트(연결 영상 없음)는 모먼트 자체 `date`.
- 사진·포스트의 `YYMMDD`는 각 자체 `date`(과거 연도) 기준. 예: `2022-07-14` → `220714`.

## 5. 그룹 / 배칭 / 순서

- **그룹 키 = (그룹 제목 + 날짜)**. 같은 그룹 미디어끼리 한데 묶고, 다른 그룹은 다른 트윗.
  "그룹 제목"은 종류별로 다르며, 표시 텍스트(§4)와는 별개의 그룹 판별용 값이다:
  - 사진: subfix 제거한 제목
  - 모먼트: 상위 영상 제목 (`video_id`로 참조. 상위 영상 없으면 모먼트 자체 `title`)
  - 포스트: 포스트 제목
- 그룹 내 배칭 규칙 (**현재 구현: 최대 배칭**):
  - 종류(이미지/영상) 상관없이 **최대 4개 미디어 / 1트윗** (`group.ts`의 `MAX_MEDIA_PER_TWEET`)
  - 영상 여러 개, 이미지+영상 혼합도 한 트윗에 묶음
- **순서**: 전체를 **오래된 날짜부터**(오름차순) 정렬. 리드 트윗 없이 첫 미디어부터 바로 체인.
- 쓰레드: 각 트윗은 직전 트윗에 답글(`in_reply_to_tweet_id`)로 연결.

> 참고(홈 화면과 방향 반대): 홈 "그해 오늘"(`OnThisDay.tsx`)은 최신 연도 먼저 표시. 봇 쓰레드는 오래된 순.

### ⚠️ 미확정(실측으로 확인 — Task 12)

X 앱 UI에는 2024년 "mixed media" 기능으로 한 게시물에 사진+영상 혼합/영상 다중(최대 4)이 가능하다.
공개 API가 이를 지원하는지는 3자 자료로 확정되지 않았다(자료상 "이미지4 OR 영상1").

- **현재 구현은 최대 배칭(4개/트윗, 혼합·다중 허용)** 을 기본으로 한다(사용자 요청).
- API 키 확보 후 실측에서 4영상/혼합 트윗이 실제로 게시되는지 확인한다.
- **미지원으로 판명되면** `planTweets`를 종류별 배칭(이미지 4 / 영상 1, 혼합 금지)으로 되돌린다(git 이력에 이전 버전 있음).
- `postThread`가 트윗별 에러 격리를 하므로, 배칭 트윗이 거부돼도 크래시 없이 스킵된다(해당 미디어는 그 회차 미게시).

## 6. 미디어 업로드

- R2에서 `GetObject`로 파일 다운로드 (기존 `scripts/remux-r2-videos.ts` 패턴).
- X 업로드는 `twitter-api-v2` 라이브러리 사용 (OAuth 1.0a, 청크 업로드, 쓰레드 지원).
  - 영상: 청크 업로드(INIT/APPEND/FINALIZE) + 처리 완료 폴링.
  - 이미지: simple 업로드.
- 업로드로 얻은 `media_id`를 트윗 게시 시 첨부.

## 7. 중복 방지

- Supabase에 `tweet_bot_log` 테이블 신설: `run_date`(`YYYY-MM-DD`, unique), `posted_at`, `tweet_count`.
- 실행 시 오늘 `run_date`가 이미 있으면 **스킵**(같은 날 크론 재실행/수동 재실행 방어).
- 키가 **오늘 전체 날짜**이므로 내년 같은 MM-DD는 다른 키 → 정상적으로 다시 게시된다.

## 8. 실행 / 스케줄

- 코드: `scripts/dailyTweet.ts` (기존 스크립트처럼 `tsx`로 실행, `process.env.VITE_*` 사용).
- GitHub Actions 크론: `0 15 * * *` (UTC) = **KST 00:00**. 워크플로에 `TZ=Asia/Seoul` 지정하여
  "오늘 날짜"를 KST로 계산.
- 시크릿은 GitHub Secrets에 저장.

## 9. 인증 / 비용

- 인증: X OAuth 1.0a User Context (API key/secret + access token/secret 4개). v1.1 미디어 업로드에 필수.
- 비용: 종량제 $0.015/트윗 (URL 미포함). ~5트윗/일 × 30일 ≈ **월 $2.25 (~3,000원)**. 크레딧 선불 충전 필요.

## 10. 에러 처리

- 개별 미디어(다운로드/업로드) 실패 시 해당 항목만 스킵하고 계속 진행.
- 실행 종료 시 게시/스킵/실패 건수 리포트를 로그로 남김.
- 영상이 X 스펙(용량/길이) 초과 시 스킵 + 로그.

## 11. 나중에 확인할 항목 (open)

1. 영상 다중/혼합 배칭 실제 가능 여부 → API 키 확보 후 테스트.
2. 쓰레드 길이 상한(지저분함 대책) → 실제 하루치 테스트 게시 결과 보고 판단. **현재는 상한 없이 전부 게시.**
3. X 영상 스펙 한도(용량/길이/포맷) → 구현 시 확인, 초과분 스킵 처리.
4. X 개발자 콘솔 최소 크레딧 충전액.
